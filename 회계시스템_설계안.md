## 2. DB 설계 (Prisma 모델)

### 2-1. 회계단위 (AccUnit)
> 남정교회: 없음 (코드에 내포) / jesuson: 회계단위 테이블

```prisma
model AccUnit {
  id        Int      @id @default(autoincrement())
  code      String   @unique @db.VarChar(10)    // "001", "002" 등
  name      String   @db.VarChar(50)            // "일반회계", "건축회계" 등
  sortOrder Int      @default(0)
  isActive  Boolean  @default(true)
  createdAt DateTime @default(now())

  accounts  AccAccount[]
  vouchers  AccVoucher[]
  closings  AccClosing[]
  balances  AccBalance[]

  @@map("acc_units")
}
```

### 2-2. 계정과목 (AccAccount)
> 남정교회: TB_ACCODE (6자리, 3단계) / jesuson: 4단계 트리

```prisma
model AccAccount {
  id        Int      @id @default(autoincrement())
  unitId    Int                                  // 회계단위
  unit      AccUnit  @relation(fields: [unitId], references: [id])
  code      String   @db.VarChar(10)            // "110010" 등
  name      String   @db.VarChar(100)           // "십일조헌금"
  type      String   @db.VarChar(1)             // "D"=수입, "C"=지출
  parentId  Int?                                 // 상위 계정 (트리 구조)
  parent    AccAccount? @relation("AccTree", fields: [parentId], references: [id])
  children  AccAccount[] @relation("AccTree")
  level     Int      @default(0)                // 0=대분류, 1=중분류, 2=소분류
  sortOrder Int      @default(0)
  isActive  Boolean  @default(true)
  description String? @db.VarChar(200)          // 설명 (jesuson 참고)
  createdAt DateTime @default(now())

  voucherItems AccVoucherItem[]

  @@unique([unitId, code])
  @@index([unitId, type])
  @@index([parentId])
  @@map("acc_accounts")
}
```

### 2-3. 전표 (AccVoucher) + 전표항목 (AccVoucherItem)
> 남정교회: TB_ACMA (한 전표에 여러 계정) / jesuson: 결의서 + 전표

```prisma
// 전표 헤더 (날짜 + 전표번호 단위)
model AccVoucher {
  id          Int      @id @default(autoincrement())
  unitId      Int                                // 회계단위
  unit        AccUnit  @relation(fields: [unitId], references: [id])
  voucherNo   String   @db.VarChar(20)           // "20260405-001" (YYYYMMDD-순번)
  type        String   @db.VarChar(1)            // "D"=수입전표, "C"=지출전표
  date        DateTime @db.Date                  // 거래일자
  description String?  @db.VarChar(200)          // 전표 적요
  totalAmount Int      @default(0)               // 합계금액
  isClosed    Boolean  @default(false)           // 마감 여부
  createdBy   String   @db.VarChar(50)           // 작성자
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  items       AccVoucherItem[]

  @@unique([unitId, voucherNo])
  @@index([unitId, date])
  @@index([date])
  @@map("acc_vouchers")
}

// 전표 항목 (한 전표에 여러 행)
model AccVoucherItem {
  id          Int      @id @default(autoincrement())
  voucherId   Int
  voucher     AccVoucher @relation(fields: [voucherId], references: [id], onDelete: Cascade)
  seq         Int                                // 행 순번
  accountId   Int                                // 계정과목
  account     AccAccount @relation(fields: [accountId], references: [id])
  amount      Int                                // 금액
  description String?  @db.VarChar(200)          // 적요 (행별)
  counterpart String?  @db.VarChar(100)          // 거래처/납부자
  createdAt   DateTime @default(now())

  @@index([voucherId])
  @@index([accountId])
  @@map("acc_voucher_items")
}
```

### 2-4. 마감 (AccClosing)
> 월별 마감 처리

```prisma
model AccClosing {
  id           Int      @id @default(autoincrement())
  unitId       Int
  unit         AccUnit  @relation(fields: [unitId], references: [id])
  year         Int                               // 회계연도
  month        Int                               // 월 (1~12)
  totalIncome  Int      @default(0)              // 당월 수입 합계
  totalExpense Int      @default(0)              // 당월 지출 합계
  carryOver    Int      @default(0)              // 이월잔액
  closedAt     DateTime?                         // 마감일시 (null=미마감)
  closedBy     String?  @db.VarChar(50)          // 마감자

  @@unique([unitId, year, month])
  @@map("acc_closings")
}
```

### 2-5. 이월잔액 (AccBalance)
> 남정교회: TB_BAS / jesuson: 회계연도/이월

```prisma
model AccBalance {
  id        Int      @id @default(autoincrement())
  unitId    Int
  unit      AccUnit  @relation(fields: [unitId], references: [id])
  year      Int                                  // 회계연도
  amount    Int      @default(0)                 // 전기이월 잔액

  @@unique([unitId, year])
  @@map("acc_balances")
}
```

### 2-6. User 확장
```prisma
// User 모델에 추가
accountAccess Boolean @default(false)  // 회계 접근 권한
```

## 3. 전표번호 규칙

| 구분 | 형식 | 예시 |
|------|------|------|
| 남정교회 | Date + Slno | 20260405 + 001 |
| jesuson | YYYYMMDD-순번 | 20260405-001 |
| **동천교회** | **YYYYMMDD-순번** | **20260405-001** |

- 같은 날짜 내에서 자동 순번 증가
- 수입/지출 구분 없이 통합 번호

## 5. 데이터 흐름

```
[계정과목 설정] → [회계단위 설정] → [이월잔액 입력]
                                        │
                    ┌───────────────────┘
                    ▼
              [전표입력] ──→ [전표현황] ──→ [보고서]
              (수입/지출)    (조회/수정)    (월별/계정별/일자별)
                    │
                    ▼
              [월 마감] ──→ [결산현황]
              (수정잠금)     (이월 계산)
```

### 마감 프로세스
1. 관리자가 "4월 마감" 실행
2. 4월 전표 수정/삭제 잠금 (isClosed=true)
3. 4월 수입/지출 합계 계산 → AccClosing 저장
4. 5월 이월잔액 = 4월 이월 + 4월 수입 - 4월 지출
5. 필요 시 마감 해제 가능 (관리자만)

## 7. 남정교회 vs jesuson vs 동천교회 비교

| 항목 | 남정교회 (VB6) | jesuson.net | 동천교회 (설계) |
|------|--------------|-------------|--------------|
| 계정코드 | 6자리, 3단계 | 4자리, 4단계 | 6자리, 3단계 (트리) |
| 회계단위 | 코드에 내포 | 별도 테이블 | **별도 테이블** |
| 전표 구조 | 헤더+다중행 | 결의서+다중행 | **헤더+다중행** |
| 전표번호 | YYYYMMDD+순번 | YYYYMMDD-순번 | **YYYYMMDD-순번** |
| 예산 관리 | TB_PLAN | 있음 (통제) | 2차 구현 |
| 결재라인 | 없음 | 6단계 | 2차 구현 |
| 이월잔액 | TB_BAS | 있음 | **별도 테이블** |
| 거래처 | TB_CUST | 있음 | 전표 항목에 텍스트 |
| 통장 관리 | TB_TONGJANG | 있음 | 2차 구현 |
| 보고서 | Crystal Reports 60+ | 다양 | **월별/계정별/일자별** |
| 엑셀 | 없음 | 있음 | 1차 구현 |

## 8. 구현 계획

### Phase 1: 기반 + 설정 (계정과목/회계단위)
- Prisma 모델 생성 (AccUnit, AccAccount, AccVoucher, AccVoucherItem, AccClosing, AccBalance)
- User 모델에 accountAccess 추가
- Header에 "장부관리" 버튼 추가
- /accounting 레이아웃 (사이드바 메뉴)
- 회계단위 관리 페이지
- 계정과목 관리 페이지 (트리 UI)
- 남정교회 기본 계정과목 시드 데이터

### Phase 2: 전표입력/조회
- 전표입력 (수입/지출 탭, 다중행 입력)
- 전표번호 자동 채번
- 전표현황 (검색/수정/삭제)
- 마감된 전표 수정 잠금

### Phase 3: 보고서/결산/마감
- 월별 수입지출 보고서
- 계정별 현황
- 일자별 현황
- 결산현황 (이월 계산)
- 마감/해제
- 엑셀 내보내기

### Phase 4: 확장 (2차)
- 예산 관리/통제
- 통장(계좌) 관리
- 거래처 관리
- 결재라인
- 연간 결산 보고서
